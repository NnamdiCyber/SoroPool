# Security Policy

## Reporting a Vulnerability

SoroPool takes the security of its smart contracts and infrastructure seriously.

**Do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via one of the following channels:

- **Smart contract / protocol bugs**: Submit a report via [Immunefi](https://immunefi.com/bounty/soropool)
- **Infrastructure / backend / frontend issues**: Email **security@soropool.finance**

We aim to acknowledge receipt within 24 hours and provide a detailed response
within 72 hours, including an assessment and expected fix timeline.

## Scope

- All smart contracts under `contracts/`
- Backend API server under `backend/`
- Frontend application under `frontend/`
- Infrastructure configuration under `infrastructure/`

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| latest  | :white_check_mark: |
| < latest| :x:                |

## Disclosure

We ask that you give us a reasonable time to fix the issue before any public
disclosure. We will coordinate disclosures with you once a fix is released.
